# Triage Backlog

The intake list. `/triage` drains the **Open** section: it picks the top item and routes it by
`(type)`. A `(bug)` or `(data)` item is fixed end-to-end and moves straight to **Done**. A `(story)`
that gets turned into a queued spec moves to **Specced (in queue)** â€” it has left the backlog for the
spec queue and is no longer re-triageable â€” and only reaches **Done** when `/implement-next` ships
that spec. Each item lives in exactly one state at a time (never Open *and* specced).

**Format:** `- [ ] (type) Title â€” one-line context [area]`
**Optional model tag:** `[model: fable|opus|sonnet, effort: xhigh|high|...]` â€” a routing hint for judgment-heavy items. Fix loops run in the MAIN session, so the dispatcher should surface the tag ("this item recommends X â€” /model to switch, or proceed") rather than silently ignore it. Specs carry the same hint as `model:`/`effort:` frontmatter, which `/implement-next` passes to the implementer agents automatically.
**Types & routing:**
- `(bug)` â†’ `fundscore-data:fix-bug` (fundscore-harness plugin) â€” lean loop: reproduce â†’ fix â†’ verify â†’ review (Claude + codex) â†’ commit.
- `(data)` â†’ `fundscore-data:fix-data` (fundscore-harness plugin) â€” lean loop: root-cause at source â†’ fix â†’ rebuild â†’ /check-data + data-reviewer â†’ review (Claude + codex) â†’ commit. (No synthetic data; fix upstream.)
- `(story)` â†’ `spec-story`: clarity-gate â†’ a lean spec in `specs/queue/` (or escalate to the owner if vague). Discovery-heavy, net-new product direction goes to `/critique-funds` â†’ review-proposals â†’ `/spec-approved` instead. Once a spec is queued, the item moves to **Specced (in queue)** (below), not Done.

Keep items small and single-purpose. Put the type first so the dispatcher can route without reading the whole line.
