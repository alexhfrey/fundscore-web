# Triage Backlog

One list of everything to do. `/triage` drains it: picks the top **Open** item, routes by
`(type)` to the matching loop, and on success checks it off and moves it to **Done**.

**Format:** `- [ ] (type) Title ‚Äî one-line context [area]`
**Optional model tag:** `[model: fable|opus|sonnet, effort: xhigh|high|...]` ‚Äî a routing hint for judgment-heavy items. Fix loops run in the MAIN session, so the dispatcher should surface the tag ("this item recommends X ‚Äî /model to switch, or proceed") rather than silently ignore it. Specs carry the same hint as `model:`/`effort:` frontmatter, which `/implement-next` passes to the implementer agents automatically.
**Types & routing:**
- `(bug)` ‚Üí `/fix-bug` ‚Äî lean loop: reproduce ‚Üí fix ‚Üí verify ‚Üí review (Claude + codex) ‚Üí commit.
- `(data)` ‚Üí `/fix-data` ‚Äî lean loop: root-cause at source ‚Üí fix ‚Üí rebuild ‚Üí /check-data + data-reviewer ‚Üí review (Claude + codex) ‚Üí commit. (No synthetic data; fix upstream.)
- `(story)` ‚Üí existing feature pipeline: `/critique-funds` ‚Üí review-proposals ‚Üí `/spec-approved` ‚Üí `/implement-next`.

Keep items small and single-purpose. Put the type first so the dispatcher can route without reading the whole line.
